# Layered 2.5D Avatar Redesign

**Date**: 2026-03-26
**Status**: Approved
**Goal**: Replace flat image-filter portrait mode with a layered compositing system that produces expressive, 3D-feeling avatar animation in the terminal.

## Problem

Portrait mode applies PIL image filters (brightness, scanlines, region darkening) to a static image. The result:
- States are visually ambiguous — idle and thinking look nearly identical
- Speaking produces no visible mouth movement
- No real facial animation — just filter parameter changes
- Feels flat and lifeless despite rich underlying state machine

The cyberpunk hand-crafted frame set has distinct states but is low-fidelity ASCII art.

## Solution

Decompose the avatar face into depth-ordered transparent layers. Pre-composite layer combinations into sixel-encoded frames at startup. At runtime, the avatar indexes into pre-baked frames for zero-CPU animation with 3D parallax depth.

## Constraints

- **Rendering target**: Sixel-capable terminals (GNOME Terminal 44+, foot, WezTerm, kitty)
- **Runtime cost**: Near-zero CPU — pure frame lookup from pre-rendered atlas
- **Art style**: Ghost in the Shell inspired cyberpunk — sharp features, tech-augmented, GITS color palette
- **Asset creation**: AI-generated from reference image, not hand-drawn
- **Backward compatibility**: Existing cyberpunk and portrait frame sets remain as fallbacks

## Layer Architecture

8 depth-ordered layers, each with pre-rendered variants as transparent PNGs at a fixed canvas size (512x512):

| Layer | Depth | Variants | Purpose |
|-------|-------|----------|---------|
| Background | 0 (farthest) | 3 | Ambient glow intensity (idle dim, thinking pulse, error red) |
| Hair/back | 1 | 3 | Position shift per head angle |
| Face shape | 2 | 5 | Head angle: center, left-15deg, right-15deg, up-10deg, down-10deg |
| Eyes | 3 | 15 | 5 directions (center/L/R/up/down) x 3 states (open/half/closed) |
| Eyebrows | 4 | 4 | Neutral, raised, furrowed, asymmetric |
| Nose | 5 | 3 | Matches head angle (center/left/right) |
| Mouth | 6 | 6 | Closed, slight, open, wide, smile, glitch |
| Overlay FX | 7 (nearest) | 8 | Scanlines, CRT bloom, hologram flicker, chromatic aberration, glitch corruption |

**Total layer PNGs**: 47

### Parallax System

When the head "rotates", layers shift horizontally by different pixel amounts based on depth:

- Background: +/-12px
- Hair: +/-8px
- Face: +/-5px
- Eyes/brows: +/-2px
- Mouth/nose: +/-1px

This creates convincing depth from flat layers without 3D geometry.

## State-to-Frame Mapping

Each state selects a subset of valid layer combinations:

### Idle (~30 frames, 0.8s rate)
- Head: center, occasional subtle drift L/R
- Eyes: open with blink cycle, rare glance to side
- Brows: neutral
- Mouth: closed
- Overlay: dim scanlines, breathing glow pulse

### Thinking (~20 frames, 0.15s rate)
- Head: slow drift L to center to R to center
- Eyes: dim/half-closed, pupil wanders L/R/up
- Brows: furrowed
- Mouth: closed
- Overlay: intense scanline sweep, chromatic aberration pulses

### Speaking (~24 frames, 0.1s rate)
- Head: center, subtle nod (up/down cycle)
- Eyes: open, steady
- Brows: neutral/raised
- Mouth: cycles closed to slight to open to wide to open to slight
- Overlay: light scanlines
- Frame composition: 4 mouth shapes x 2 head positions x 3 overlay phases

### Listening (~12 frames, 0.4s rate)
- Head: slight tilt (asymmetric), attentive
- Eyes: wide open, bright
- Brows: raised
- Mouth: slight smile
- Overlay: bright glow, wide scanlines

### Error (~16 frames, 0.2s rate)
- Head: jitter (random angle per frame)
- Eyes: dead/glitch (misaligned between frames)
- Brows: furrowed
- Mouth: glitch
- Overlay: heavy corruption, red tint, noise bands, chromatic aberration

### Micro-Events (overlaid during idle/listening)
- **Blink**: eye layer swaps open-half-closed-half-open (4 frames)
- **Glitch burst**: overlay cycles through corruption variants (6 frames)
- **Flicker**: overlay switches to hologram dropout (3 frames)
- **Glance**: eye layer shifts direction briefly (2 frames)

**Total pre-rendered frames**: ~117 unique composited frames stored as sixel-encoded strings.

## Asset Pipeline

### Directory Structure

```
assets/layers/
  background/     -> bg_dim.png, bg_pulse.png, bg_error.png
  hair/           -> hair_center.png, hair_left.png, hair_right.png
  face/           -> face_center.png, face_left15.png, face_right15.png, face_up10.png, face_down10.png
  eyes/           -> eyes_{direction}_{state}.png (15 files)
  eyebrows/       -> brows_neutral.png, brows_raised.png, brows_furrowed.png, brows_asymmetric.png
  nose/           -> nose_center.png, nose_left.png, nose_right.png
  mouth/          -> mouth_closed.png, mouth_slight.png, mouth_open.png, mouth_wide.png, mouth_smile.png, mouth_glitch.png
  overlay/        -> scanline_light.png, scanline_heavy.png, crt_bloom.png, holo_flicker.png, chrom_aberr.png, glitch_corrupt.png, noise_bands.png, red_tint.png
```

### Build Flow

```
Layer PNGs (shipped with package, ~5-10MB)
    |
    v
Startup compositor (PIL)
    - Reads state-to-frame mapping config (YAML/dict)
    - For each valid layer combination:
        1. Stack layers bottom-to-top with parallax offsets
        2. Apply GITS color grade (cyan highlights, violet midtones, purple shadows)
        3. Encode composited image to sixel
        4. Store in frame dict keyed by state
    - Cache composited sixel strings to ~/.cache/ascii-avatar/frames/
    |
    v
Runtime: pure dict lookup — frames[state][index]
```

### Cache

- Location: `~/.cache/ascii-avatar/frames/{hash}/`
- Hash computed from: layer file checksums + terminal pixel dimensions
- Invalidated when: layers change, terminal resizes significantly, or manually cleared
- Size: ~30-50MB (sixel strings are verbose but compress well)
- First launch build time: ~5-15 seconds
- Subsequent launches: instant (read from cache)

## Asset Generation

AI-generated layer assets. The generation script uses a two-phase approach:

**Phase 1 — Base face + structural layers (procedural + segmentation)**
1. Start from a reference face image (user-provided or from bundled default)
2. Use PIL-based face region extraction (luminance thresholding + contour detection) to separate hair, face outline, and nose into layers
3. Generate head angle variants by applying affine transforms (rotation + perspective warp) to the base face
4. Generate background and overlay layers procedurally in PIL (gradients, noise, scanline patterns)

**Phase 2 — Expression layers (AI-generated)**
1. Generate eyes, eyebrows, and mouth variants using a local image generation model (Stable Diffusion XL via diffusers, or FLUX if available)
2. Use inpainting: mask the relevant face region, generate variants with expression-specific prompts while keeping the surrounding face consistent
3. Extract the generated region as a transparent PNG at the original canvas position
4. If no local GPU is available, fall back to hand-drawn geometric shapes (circles for eyes, arcs for brows, rectangles for mouth) as placeholders

The generation script is a dev-time tool, not a runtime dependency. Generated assets are committed to the repo and shipped with the package.

Output: populated `assets/layers/` directory ready for the compositor.

## Frame Dict Shape

The compositor outputs a frame dict matching the existing interface:

```python
frames: dict[str, list[str]] = {
    "idle": [sixel_string, ...],       # ~30 frames
    "thinking": [sixel_string, ...],   # ~20 frames
    "speaking": [sixel_string, ...],   # ~24 frames
    "listening": [sixel_string, ...],  # ~12 frames
    "error": [sixel_string, ...],      # ~16 frames
    "blink": [sixel_string, ...],      # 4 frames (micro-event)
    "glitch": [sixel_string, ...],     # 6 frames (micro-event)
    "flicker": [sixel_string, ...],    # 3 frames (micro-event)
}

rates: dict[str, float] = {
    "idle": 0.8,
    "thinking": 0.15,
    "speaking": 0.1,
    "listening": 0.4,
    "error": 0.2,
}
```

This is identical to the shape returned by `cyberpunk.py` and `converter.py`, so the renderer, compositor, and state machine require zero changes.

## Success Criteria

1. Each state is **unmistakably distinct** at a glance — no ambiguity between idle/thinking/speaking
2. Speaking has **visible mouth movement** that correlates with actual speech
3. Head angle shifts create a **convincing depth/parallax effect**
4. Error state is **visually alarming** — red, glitched, broken
5. Idle feels **alive** — breathing, blinking, occasional glance
6. First launch cache build completes in **under 15 seconds**
7. Runtime frame lookup adds **no measurable CPU overhead**
8. Existing cyberpunk and portrait modes **continue to work unchanged**
